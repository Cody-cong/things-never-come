# GoodsNeverCome（购物版多巴胺网站）Spec

## Why
韩国 FoodNeverComes 等"多巴胺网站"爆火，通过模拟点外卖全流程但不真正送达，让用户获得消费仪式感却不花钱。用户希望复刻一个"购物版"——模拟网购全流程（逛店铺、加购物车、下单、追踪虚拟快递），但商品永远不会到达，最后用幽默方式结算"省下的钱与避免的冲动消费"，作为零成本的情绪出口。

## What Changes
- 新建一个移动端优先的 Web 应用（GoodsNeverCome），完整复刻电商购物体验的仪式感
- 店铺/商品浏览：分类、店铺列表、商品详情、评分评论
- 购物车与下单：加购、改数量、虚拟下单（不涉及真实支付）
- 虚拟快递追踪：地图上动画展示快递员移动轨迹 + ETA 倒计时
- "送达"结算页：弹窗提示已送达（实则无货），结算省下的金额、避免的冲动消费次数、节省的时间等幽默数据
- 订单历史：本地记录本次会话的"虚拟订单"
- 全部使用本地 Mock 数据，无后端、无真实交易

## Impact
- Affected specs: 无（全新项目，空目录）
- Affected code: 全新代码库，位于项目根目录
- 技术栈：Next.js 14 (App Router) + TypeScript + Tailwind CSS；状态用 React Context + useReducer；数据用本地 JSON/TS 常量；地图用 SVG 动画轨迹（不依赖第三方地图服务，保持简单零密钥）

## ADDED Requirements

### Requirement: 商品浏览与分类
系统 SHALL 提供移动端优先的电商首页，展示用户问候、搜索框、分类标签与商品网格，点击分类切换商品列表，点击商品进入详情页。

#### Scenario: 浏览首页
- **WHEN** 用户打开网站
- **THEN** 看到顶部「Hi + 用户名」问候语、可输入搜索框、分类标签横滑（第一个为「全部」）、当前分类的商品网格列表，带骨架屏加载动效

#### Scenario: 搜索商品
- **WHEN** 用户在搜索框输入关键词并回车
- **THEN** 跳转到搜索结果页，展示匹配的商品列表

#### Scenario: 切换分类
- **WHEN** 用户点击某个分类标签
- **THEN** 下方商品网格实时更新为该分类下的商品
- **WHEN** 分类下无商品
- **THEN** 显示空状态友好提示

#### Scenario: 查看商品详情
- **WHEN** 用户点击某个商品卡片
- **THEN** 进入商品详情页，展示大图、价格、规格、评价

### Requirement: 商品详情与加购
系统 SHALL 提供商品详情页，展示大图、价格、规格、评价，可加入购物车。

#### Scenario: 查看商品并加购
- **WHEN** 用户点击商品
- **THEN** 展示商品大图、标题、价格、规格选项、累计评价
- **WHEN** 用户点击"加入购物车"
- **THEN** 购物车角标数字 +1，并出现轻量动效反馈

### Requirement: 购物车与虚拟下单
系统 SHALL 提供购物车页面，可调整数量、删除、合计金额，并完成虚拟下单。

#### Scenario: 提交订单
- **WHEN** 用户在购物车点击"结算"
- **THEN** 跳转订单确认页（地址为虚构默认地址、金额合计）
- **WHEN** 用户点击"提交订单"
- **THEN** 进入快递追踪页，购物车清空，订单写入本地历史

### Requirement: 虚拟快递追踪
系统 SHALL 在下单后展示快递员在地图上的动画移动轨迹与 ETA 倒计时，但快递永远不会真正到达。

#### Scenario: 追踪快递
- **WHEN** 订单提交后
- **THEN** 展示 SVG 地图，快递员图标沿路径移动，ETA 倒计时递减，状态依次变为"已接单→已取货→配送中→即将送达"
- **WHEN** ETA 归零时
- **THEN** 触发"送达"结算页

### Requirement: 送达结算页（核心幽默点）
系统 SHALL 在"送达"时弹出结算页，告知用户商品已送达（实则无货），并展示本次省下的金额、避免的冲动消费、节省的时间等幽默统计。

#### Scenario: 查看结算
- **WHEN** 快递"送达"
- **THEN** 展示"您的商品已送达（并没有）"提示，列出：省下金额、避免冲动消费次数、节省的决策时间、虚拟碳足迹减少
- **WHEN** 用户点击"再来一单"
- **THEN** 返回首页继续模拟

### Requirement: 订单历史
系统 SHALL 在本地记录本次会话的虚拟订单列表，可查看历史"未送达"订单。

#### Scenario: 查看历史
- **WHEN** 用户从首页进入"我的订单"
- **THEN** 展示历史订单卡片（订单号、商品缩略、金额、状态均为"已送达(并没有)"）

### Requirement: 移动端优先与视觉风格
系统 SHALL 以移动端优先设计，视觉风格贴近真实电商 App（卡片、阴影、底部导航栏），左侧/底部仅图标导航。

#### Scenario: 响应式
- **WHEN** 在手机宽度访问
- **THEN** 单列布局、底部图标导航栏（首页、分类、购物车、订单、我的）
- **WHEN** 在桌面宽度访问
- **THEN** 居中展示手机宽度容器，两侧留白，保持移动 App 观感

## 非目标（明确不做）
- 不做真实支付、真实登录、真实物流
- 不做后端 API 与数据库
- 不做真实的第三方地图（用 SVG 动画代替）
- 不做用户系统与持久化（仅会话内 localStorage 可选）
