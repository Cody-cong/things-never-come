# things never come（购物版多巴胺网站）Spec

## Why

韩国 FoodNeverCome 等"多巴胺网站"爆火，通过模拟点外卖全流程但不真正送达，让用户获得消费仪式感却不花钱。用户希望复刻一个"购物版"——模拟网购全流程（浏览商品、加购物车、下单），但商品永远不会到达，最后生成一张可以保存的虚拟账单，作为零成本的情绪出口。

## What Changes

- 新建一个移动端优先的 Web 应用（things never come），完整复刻电商购物体验的仪式感
- 商品浏览：分类、商品网格、商品详情
- 购物车与下单：加购、改数量、删除、虚拟下单（不涉及真实支付）
- 虚拟账单：提交订单后生成一张精美账单，可打印或保存为 PDF
- 管理后台：管理员可添加/编辑/删除商品与分类，并设置哪些商品出现在首页热门
- 全部使用本地 Mock 数据，无后端、无真实交易

## Impact

- Affected specs: 无（全新项目，空目录）
- Affected code: 全新代码库，位于项目根目录
- 技术栈：Next.js 14 (App Router) + TypeScript + Tailwind CSS；状态用 React Context + useReducer；数据用本地 JSON/TS 常量 + localStorage

## ADDED Requirements

### Requirement: 商品浏览与分类

系统 SHALL 提供移动端优先的电商首页，展示用户问候、搜索框、热门商品与分类入口，点击商品进入详情页。

#### Scenario: 浏览首页

- **WHEN** 用户打开网站
- **THEN** 看到 Hero 区品牌标语、可输入搜索框、以及最多 6 个热门商品的两列网格，带骨架屏加载动效

#### Scenario: 查看热门商品

- **WHEN** 用户在首页浏览热门商品
- **THEN** 商品以两列网格展示，不显示分类标签
- **WHEN** 用户点击"View more"
- **THEN** 跳转分类页查看全部商品

#### Scenario: 搜索商品

- **WHEN** 用户在搜索框输入关键词并回车
- **THEN** 跳转到搜索结果页，展示匹配的商品列表

#### Scenario: 切换分类

- **WHEN** 用户进入分类页并点击某个分类标签
- **THEN** 下方商品网格实时更新为该分类下的商品
- **WHEN** 分类下无商品
- **THEN** 显示空状态友好提示

#### Scenario: 查看商品详情

- **WHEN** 用户点击某个商品卡片
- **THEN** 进入商品详情页，展示大图、价格、规格、描述

### Requirement: 商品详情与加购

系统 SHALL 提供商品详情页，展示大图、价格、规格、描述，可加入购物车。

#### Scenario: 查看商品并加购

- **WHEN** 用户点击商品
- **THEN** 展示商品大图、标题、价格、规格选项、商品描述
- **WHEN** 用户点击"加入购物车"
- **THEN** 购物车角标数字 +1，并出现轻量动效反馈

### Requirement: 购物车与虚拟下单

系统 SHALL 提供购物车页面，可调整数量、删除、合计金额，并完成虚拟下单。

#### Scenario: 提交订单

- **WHEN** 用户在购物车点击"结算"
- **THEN** 跳转订单确认页，展示商品清单与金额合计
- **WHEN** 用户点击"提交订单"
- **THEN** 购物车清空，订单写入本地历史，并进入账单页

### Requirement: 虚拟账单

系统 SHALL 在提交订单后立即生成一张精美的虚拟账单，用户可以保存或打印。

#### Scenario: 查看账单

- **WHEN** 订单提交后
- **THEN** 展示包含订单号、时间、商品明细、合计金额的账单
- **WHEN** 用户点击"保存账单"
- **THEN** 唤起浏览器打印，可保存为 PDF

### Requirement: 管理后台

系统 SHALL 提供管理员入口，管理商品、分类、FAQ 与用户反馈，并控制首页热门商品。

#### Scenario: 设置首页热门商品

- **WHEN** 管理员进入商品管理
- **THEN** 可为每个商品切换"热门"状态
- **THEN** 首页自动展示热门状态为 true 的商品，最多 6 个

#### Scenario: 管理商品与分类

- **WHEN** 管理员进入商品/分类管理
- **THEN** 可添加、编辑、删除商品与分类
- **THEN** 所有修改持久化到 localStorage

### Requirement: 移动端优先与视觉风格

系统 SHALL 以移动端优先设计，视觉风格贴近真实电商 App（卡片、阴影、底部导航栏），左侧/底部仅图标导航。

#### Scenario: 响应式

- **WHEN** 在手机宽度访问
- **THEN** 单列布局、底部图标导航栏（首页、购物车、我的）
- **WHEN** 在桌面宽度访问
- **THEN** 居中展示容器，两侧留白，保持移动 App 观感

### Requirement: 首次访问引导

系统 SHALL 在用户首次打开网站且无本地档案时，弹出引导弹窗让用户选择头像与名字，并提供默认选项。

#### Scenario: 首次访问设置默认档案

- **WHEN** 用户首次打开网站
- **THEN** 弹出头像与名字选择弹窗
- **THEN** 默认头像为项目内置头像图库的第二张图，默认名字为 "Zara"
- **WHEN** 用户点击"默认"
- **THEN** 使用默认名字与头像创建档案并关闭弹窗

### Requirement: 个人中心

系统 SHALL 提供个人中心页，展示当前用户的头像与名字，并允许直接修改。

#### Scenario: 修改头像和名字

- **WHEN** 用户进入"我的"页
- **THEN** 看到当前头像与名字，不显示会员等级等额外文案
- **WHEN** 用户点击头像选项或修改名字输入框
- **THEN** 实时保存变更
- **THEN** 页面不再提供"设置"入口

## 非目标（明确不做）

- 不做真实支付、真实登录、真实物流
- 不做后端 API 与数据库
- 不做虚拟快递追踪与地图动画
- 不做订单历史列表页
- 不做余额、物流、评价系统
- 不做用户系统与持久化（仅会话内 localStorage 可选）
